# database-migrations

## Overview
When you need to evolve database schema safely over time

## Process
1. Initialize migration system
2. Create migration files for schema changes
3. Run migrations in correct order
4. Handle rollbacks when needed
5. Verify migration success

## Examples
- Schema evolution for applications
- Team database development
- Rolling back problematic changes

## Limitations
Migration conflicts in team environments; requires downtime for large changes
