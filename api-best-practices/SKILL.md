# api-best-practices

## Overview
When designing or implementing REST APIs

## Process
1. Define resource-based URL patterns
2. Use appropriate HTTP methods (GET, POST, PUT, DELETE)
3. Implement consistent response formats and status codes
4. Add proper error handling and validation
5. Document API with OpenAPI/Swagger

## Examples
- Building new REST APIs
- Refactoring existing APIs
- API documentation and versioning

## Limitations
Not applicable to GraphQL or gRPC APIs
